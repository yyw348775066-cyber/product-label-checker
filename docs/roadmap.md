# 产品标签审核对比工具 Roadmap

## 已完成功能
- 标签图片上传
- 五个平台AI结果输入
- 标签字段对比表
- 差异判断
- 风险高亮
- 审核汇总
- CSV导出
- 人工备注
- 最终审核结论
- 第三方食品检验报告审核模块
- pytest测试
- GitHub版本管理

## 当前开发原则
- 不重构项目
- 小步迭代
- 不新增复杂依赖
- 优先真实案例验证
- 每次修改后运行 pytest
- 通过后再 git commit 和 git push

## 下一阶段优先功能

### 1. 审核规则命中
目标：自动识别常见标签/报告风险。

示例规则：
- 未看到生产许可证
- 未看到贮存条件
- 未看到联系方式
- 委托企业存在但委托地址缺失
- 报告缺少 CMA/CNAS 资质信息
- 报告缺少判定依据
- 检验结论未看到
- 签发日期未看到

### 2. 标签 vs 检验报告交叉核对
目标：核对标签信息和报告信息是否一致。

重点字段：
- 产品名称
- 执行标准
- 生产企业
- 委托单位
- 样品规格 / 净含量
- 签发日期 / 生产日期

### 4. 真实案例库
目标：持续沉淀真实审核案例。

路径：
- data/real_cases/

每个案例建议记录：
- 标签图片/报告来源说明
- 五个平台AI输出
- 人工最终结论
- 实际发现风险
- 后续提示词优化点

## 暂不做的事项
