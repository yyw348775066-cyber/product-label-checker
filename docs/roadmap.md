## 当前开发原则
- 不重构项目
- 小步迭代
- 不新增复杂依赖
- 优先真实案例验证
- 每次修改后运行 pytest
- 通过后再 git commit 和 git push

## 下一阶段优先功能

### 1. 审核规则命中
目标：自动识别常见标签/报告风险。

示例规则：
- 未看到生产许可证
- 未看到贮存条件
- 未看到联系方式
- 委托企业存在但委托地址缺失
- 报告缺少 CMA/CNAS 资质信息
- 报告缺少判定依据
- 检验结论未看到
- 签发日期未看到

### 2. 标签 vs 检验报告交叉核对
目标：核对标签信息和报告信息是否一致。

重点字段：
- 产品名称
- 执行标准
- 生产企业
- 委托单位
- 样品规格 / 净含量
- 签发日期 / 生产日期

### 4. 真实案例库
目标：持续沉淀真实审核案例。

路径：
- data/real_cases/

每个案例建议记录：
- 标签图片/报告来源说明
- 五个平台AI输出
- 人工最终结论
- 实际发现风险
- 后续提示词优化点

## 暂不做的事项

## DeepSeek 网页自动化 MVP

- 首次使用需运行 Playwright 浏览器安装：`python -m playwright install chromium`
- 首次打开后手动登录 DeepSeek
- 登录态保存在 `automation/browser_data/deepseek/`
- 当前阶段只打开 DeepSeek 网页并保存登录态，不自动上传、不自动提问、不抓取回答

## 当前重点方向（2026-05）

### AI 网页辅助工作流
当前采用：
- Flask 审核工作台
- Playwright 浏览器辅助
- DeepSeek 持久化登录

原则：
- 不做高风险网页机器人
- 不做 WebSocket 抓包
- 不做 iframe 注入
- 不做无人值守自动发送
- 优先稳定性

### 当前 DeepSeek 辅助能力
- 打开 DeepSeek 网页
- 持久化登录态
- 复制审核提示词
- 人工上传文件
- 人工发送问题
- 人工复制回答
- 系统自动结构化对比

### 后续可能方向
- DeepSeek 文件路径提示
- 一键复制提示词
- 多模型辅助入口
- 审核案例沉淀
- AI 一致率分析

### 暂不进行
- 五平台全自动网页机器人
- DOM 持续监听
- WebSocket 抓包
- iframe 自动收发
- 无人值守自动审核
